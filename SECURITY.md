# Security Policy

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported          |
|---------|--------------------|
| 0.1.x   | :white_check_mark: |
| &lt; 0.1 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do not** open a public GitHub issue for security-sensitive bugs.
2. Contact the maintainer(s) privately (e.g. via the repository owner’s GitHub profile or email if listed).
3. Provide a clear description of the issue, steps to reproduce, and impact if possible.
4. Allow a reasonable time for a fix before any public disclosure.

We will acknowledge your report and work on a fix. This project is intended for **research and education**; it is not a certified medical device. Do not use it as the sole basis for clinical decisions.

## Scope

Security issues include (but are not limited to): remote code execution, sensitive data exposure, authentication/authorization bypass, and dependency vulnerabilities that affect this package’s use.

Thank you for helping keep this project and its users safe.
